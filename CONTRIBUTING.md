# Guide de contribution

Merci de votre intérêt pour contribuer à Kreacity AI Stack ! Voici quelques lignes directrices pour vous aider à contribuer.

## Comment contribuer

1. **Forker le dépôt** : Créez votre propre copie du projet sur GitHub.
2. **Créer une branche** : `git checkout -b ma-fonctionnalite`
3. **Implémenter vos changements** : Suivez les conventions de code existantes.
4. **Tester** : Assurez-vous que vos changements fonctionnent comme prévu.
5. **Soumettre une pull request** : Décrivez clairement les modifications que vous avez apportées.

## Types de contributions bienvenus

- **Améliorations de la documentation**
- **Nouveaux exemples d'utilisation**
- **Optimisations de performance**
- **Support de nouveaux modèles d'embeddings**
- **Ajout de nouveaux services complémentaires**
- **Corrections de bugs**

## Standards de code

- Commentez votre code pour le rendre compréhensible
- Utilisez des noms de variables et de fonctions descriptifs
- Suivez les conventions YAML pour les fichiers docker-compose
- Testez vos modifications sur différentes plateformes si possible

## Signaler des bugs

Si vous trouvez un bug, veuillez créer une issue en incluant :
- Une description claire du problème
- Les étapes pour reproduire le bug
- Le comportement attendu et celui observé
- Votre environnement (OS, version de Docker, etc.)

## Proposer des fonctionnalités

Les propositions de nouvelles fonctionnalités sont les bienvenues ! Créez une issue pour discuter de votre idée avant de commencer à travailler dessus.

## Lignes directrices pour les pull requests

- Faites des pull requests ciblées et concises
- Incluez des tests pour les nouvelles fonctionnalités
- Mettez à jour la documentation si nécessaire
- Assurez-vous que vos modifications ne cassent pas les fonctionnalités existantes

## Licence

En contribuant, vous acceptez que vos contributions soient sous la même licence que le projet (MIT).
